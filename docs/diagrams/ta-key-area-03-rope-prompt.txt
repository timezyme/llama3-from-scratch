Title
Key Area 3: RoPE - Position Rotation on Q and K

Knowledge-transfer goal
Create a high-resolution 16:9 educational infographic for a CS265 TA-prep
explanation of RoPE in this Llama 3 inference repo. The viewer should grasp
the main idea in 10 seconds: RoPE does not add a position vector; it rotates
Q and K values in place so attention scores carry position. After 60 seconds,
the viewer should be able to explain which tensors are changed, where the code
does it, and the two Llama 3 gotchas: base 500000 and rotate-half pairing.

Core story
RoPE is a position-dependent rotation applied inside every decoder layer after
Q/K/V projection and before attention. It rotates Q for the current rows and
only the newly written K rows in the KV cache; V is cached and read by attention
but is never rotated. The cos/sin table is built once, then `forward_step`
slices it by `len_before` so prefill starts at position 0 and decode starts at
the next uncached absolute position.

Guardrail facts the image must obey but should not render unless listed
- RoPE applies to Q and K in-place before attention.
- RoPE does not append coordinates, add a learned position embedding, add a
  position column, or create a separate tensor in the visible activation path.
- For token position `p` and pair index `i`,
  `theta_i = 1 / 500000^(2i / h_d)` and `angle = p * theta_i`.
- Llama 3 uses `ROPE_BASE = 500000`, not 10000.
- Pairing is rotate-half: dimension `i` pairs with `i + h_d/2`, not even/odd
  `(0,1), (2,3)`.
- With `HEAD_DIM = 128`, `half_hd = 64`, so valid visual pairs are
  `(0,64)`, `(1,65)`, ..., `(63,127)`.
- The per-pair rotation is:
  `x_new[i] = x[i] * cos(angle) - x[i+64] * sin(angle)`
  `x_new[i+64] = x[i] * sin(angle) + x[i+64] * cos(angle)`
- Rotation applies independently to Q heads and K heads.
- Repo constants: `ROPE_BASE=500000`, `HEAD_DIM=128`, `half_hd=64`,
  `NUM_HEADS=32`, `NUM_KV_HEADS=8`, `NUM_LAYERS=32`, `S_MAX=1024`.
- Exact shape labels, if rendered, must be:
  `Q [rows,4096]`, `K new [q_seq,1024]`, `V new [q_seq,1024]`,
  `cached K [kv_seq,1024]`, `cached V [kv_seq,1024]`, `cos/sin [1024,64]`.
- `rows = batch * q_seq`.
- `kv_seq = len_before + q_seq`.
- During prefill, `q_seq` is prompt length and `len_before = 0`.
- During decode, `q_seq = 1` and the effective absolute position is
  `len_before`, not always 0.
- `kernel/rope.cu` contains `precompute_rope_table(...)`, `gpu_rope(...)`,
  and `rope_kernel(...)`.
- `src/inference_loop.cu` contains `alloc_rope_tables(...)`, which builds
  `[S_MAX, HEAD_DIM/2] = [1024,64]` cos/sin tables and uploads them to VRAM.
- `src/inference_layer.cu` contains `forward_step(...)`, computes
  `d_cos_step = d_cos_full + len_before * half_hd` and
  `d_sin_step = d_sin_full + len_before * half_hd` once before the 32-layer
  loop, then reuses those sliced pointers inside every layer.
- `rope_kernel` computes literal local position as `pos = row % q_seq`.
  The effective absolute table row comes from the already-offset
  `d_cos_step` / `d_sin_step` pointers.
- `forward_step(...)` computes Q from `d_Xnorm`, computes K/V into the
  `KVCache`, rotates `d_Q`, rotates `cache.k_at(layer, len_before, b)`, does
  not rotate V, then attention reads rotated Q plus full cached K/V.
- The exact Q call is:
  `gpu_rope(d_Q, d_cos_step, d_sin_step, rows, NUM_HEADS, HEAD_DIM, q_seq)`.
- The exact K call is:
  `gpu_rope(cache.k_at(layer, len_before, b), d_cos_step, d_sin_step, q_seq, NUM_KV_HEADS, HEAD_DIM, q_seq)`.
- Old cached K rows were rotated when first written. During decode, only the
  new K row is rotated before attention reads the full cache.
- V rows are cached and read by attention across `[0, kv_seq)`, but V rows are
  never passed to `gpu_rope`.
- `cache.advance(q_seq)` happens after all 32 decoder layers finish.
- Draw `cache.advance` outside the repeated layer loop, after all 32 layers, not
  as an operation that runs after every single layer.
- If attention scores are rendered, use only per-head GQA scoring:
  `Q_i [q_seq,128] * K_g^T [128,kv_seq] -> S_i [q_seq,kv_seq]`, scaled by
  `1/sqrt(HEAD_DIM)`.
