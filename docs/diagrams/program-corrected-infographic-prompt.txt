Main panel 3, green, title:
3. KV Cache Allocation
subtitle: `(src/kv_cache.cu)`
Content:
- Allocated before `forward_step`
- Lives in GPU VRAM
- Stores Key and Value matrices for every decoder layer
- `S_MAX = 1024`
- Used by multi-token decode to reuse previous context instead of recomputing old K/V
- In prefill, the cache is populated for the full prompt
- In decode, one new K row and one new V row are appended per generated token

1. `RMSNorm` - stabilizes/scales each token vector; epsilon `1e-5`

Main panel 5, green, title:
5. Decode Phase - Autoregressive Loop
subtitle:
Runs when more than one token is requested
Content as lettered steps:
A. Take the final hidden state from prefill or previous decode step
B. CPU `compute_lm_head_logits()` projects through `global/lm_head_weight.bin`
C. Produce logits over full vocabulary `VOCAB_SIZE = 128256`
D. Select the most likely token using greedy argmax
E. Embed the generated token and run `forward_step(q_seq = 1)`
F. Append one new K/V row to the KV cache and attend over the full cached prefix
G. Repeat until `max_new_tokens` limit or `EOT_ID = 128009`
