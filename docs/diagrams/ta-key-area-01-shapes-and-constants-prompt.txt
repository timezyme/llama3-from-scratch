Title
Key Area 1: Shapes and Constants in Llama 3 Inference

Knowledge-transfer goal
Create a high-resolution 16:9 educational infographic for CS265 TA prep. The viewer should understand the main idea in 10 seconds: tensor shapes line up because the model repeatedly returns to the same hidden width, `[s,4096]`, after temporary attention and FFN reshapes. In under 60 seconds, the viewer should be able to explain what `s`, `d`, heads, K/V sharing, `d_ff`, vocab logits, and the repo-specific `lm_head` exception mean.

Core story
Token IDs become a matrix with one row per token and 4096 columns per hidden vector. Attention splits that width into heads: Q uses 32 heads, but K/V use only 8 heads, so this Llama 3 checkpoint uses Grouped Query Attention. Attention and the FFN temporarily change the internal shape, then project back to `[s,4096]` so residual additions are valid. After 32 decoder blocks, the repo scores only the last hidden vector with a separate `[128256,4096]` `lm_head`.

Guardrail facts the image must obey but should not render unless listed
- Assignment notation:
  - `s` = sequence length, the number of tokens currently processed.
  - `d = 4096` = embedding and hidden dimension.
  - `h = 32` = query heads.
  - `h_k = 8` = key/value heads.
  - `h_d = 128` = head dimension.
  - `d = h * h_d = 32 * 128 = 4096`.
  - `d_ff = 14336` = FFN/SwiGLU intermediate width.
  - `V = 128256` = vocabulary size and logits size.
  - The decoder block is repeated `32` times.
- Repo constants from `config.h`:
  - `EMBEDDING_DIM = 4096`
  - `RMS_NORM_EPSILON = 1e-5`
  - `NUM_HEADS = 32`
  - `NUM_KV_HEADS = 8`
  - `HEAD_DIM = 128`
  - `FFN_DIM = 14336`
  - `VOCAB_SIZE = 128256`
  - `ROPE_BASE = 500000`
  - `NUM_LAYERS = 32`
- Repo internal constants from `src/inference_internal.h`:
  - `S_MAX = 1024`
  - `EOT_ID = 128009`
- `S_MAX = 1024` is this project's working cap for KV cache and RoPE table allocation. It is not the model hidden width and not the learned context length.
- The required assignment mental model is B=1, but the live repo also has batched generation. When describing live batched output, use `[batch,4096]`; when describing one next-token decision, use `x_last [4096]`.
- `forward_step` in `src/inference_layer.cu` executes the 32-layer decoder stack and returns last hidden vectors. It does not return logits or token IDs.
- Final RMSNorm happens after the 32 decoder blocks and before `x_last` is scored
  by `lm_head`; do not draw `lm_head` or final norm inside a decoder block.
- `src/inference_loop.cu` calls `forward_step`, then calls `compute_lm_head_logits`, then uses greedy argmax.
- `compute_lm_head_logits` is CPU host code in this repo, not a CUDA kernel.
- The checkpoint has `tie_word_embeddings=false`. The output head is separate from embeddings.
- `src/model_weights.cpp` loads `global/lm_head_weight.bin` as stored `[V,d] = [128256,4096]`, row-major, so logits are computed as `lm_head [V,d] @ x_last [d]`.
- If showing row-vector assignment math, label it as a math view only: `x_last [1,4096] x W_lm^T [4096,128256] -> logits [1,128256]`. Also keep the stored repo shape `[128256,4096]`.
- K/V projection width is `NUM_KV_HEADS * HEAD_DIM = 8 * 128 = 1024`.
- Q projection width is `NUM_HEADS * HEAD_DIM = 32 * 128 = 4096`.
- GQA mapping: query head `i` reads K/V group `floor(i / 4)` because `NUM_HEADS / NUM_KV_HEADS = 4`.
- A useful mini-diagram is 32 Q-head boxes grouped into 8 groups of 4, each group pointing to one K/V head.
- Attention per query head may be shown as `Q_i [s,128] x K_g^T [128,s] -> scores [s,s]`, then `weights [s,s] x V_g [s,128] -> O_i [s,128]`.
- Causal masking and softmax happen inside attention but are not the visual focus.
- RMSNorm preserves `[s,4096]` and uses epsilon `1e-5`.
- RoPE rotates Q and new K values but preserves tensor shape.
- Residual additions require both operands to be `[s,4096]`.
- FFN/SwiGLU expands each token row to `[s,14336]`, combines `SiLU(gate) * up`, then down-projects back to `[s,4096]`.
- Only the last hidden vector is scored for next-token generation; projecting all `s` rows to vocab logits would be wasteful.
- The stop token is `EOT_ID = 128009`, but it does not need to appear in this infographic unless there is room in a tiny guardrail note.
- If exact checkpoint paths appear anywhere, use only these repo-style labels:
  - `assets/llama3/dump/embeddings.bin`

  - `assets/llama3/dump/layer_00/model_layers_0_self_attn_k_proj_weight.bin`
