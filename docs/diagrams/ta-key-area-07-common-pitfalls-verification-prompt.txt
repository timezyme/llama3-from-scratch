Core story
Common pitfalls are not isolated trivia. They cluster around three places where a correct implementation can quietly become wrong: attention math/layout, weight/logit facts, and grading boundaries.

This repo avoids or exposes those mistakes with stable softmax, correct causal-mask gating, Llama 3 RoPE, GQA head mapping, transposed checkpoint weights, a separate `lm_head`, last-token logits, and reference/CUDA checks.

The student's safest TA answer is: explain the required B=1 one-token path first: one prompt, all 32 decoder layers, one output token. Then label KV cache, resident BF16 weights, and batching as repo extensions.

Guardrail facts the image must obey but should not render unless listed
- Part 2 base requirement is the B=1 one-token forward/generation path, exposed by `generate_next_token(...)` and `generate_next_token_impl(...)`.
- B=1 one-token inference still runs the full 32-layer decoder stack. Do not confuse "one generated token" with "one decoder layer".
- KV cache, resident BF16 weights, multi-token generation, batching, and extension parity tests are repo extensions. Do not imply they are mandatory professor-test requirements.
- CUDA-specific Part 2 verification belongs on a CUDA machine, specifically the L4 lane for this repo. Do not present local CPU-only macOS limits as correctness failures or milestone blockers.
- The official requirement is to verify the final argmax token against `reference.py`; `tools/verify_reference.py` compares operator outputs and final argmax against the reference path.
- Split `tests/test_m2m3_*.cpp` files include CUDA fixture checks for RMSNorm, Q/K/V, RoPE, attention, residuals, SwiGLU, decoder block, final RMSNorm, and full forward token generation.
- Stable softmax subtracts the per-row max before `exp`. The preserved formula is `exp(S[p,q] - row_max) / sum_k exp(S[p,k] - row_max)`.
- The causal mask writes `-1e6` for future positions `q > p` in square prefill. In this repo, `run_attention_heads` applies the square mask only when `q_seq == kv_seq && q_seq > 1`.
- Cached decode has `q_seq=1` and `kv_seq=len_before+1`; the new query can attend to cached positions up to itself, so there are no future cached positions in that slice.
- RoPE uses Llama 3 base `500000`, not `10000`.
- RoPE uses rotate-half pairing: index `i` pairs with `i + HEAD_DIM/2`; for `HEAD_DIM=128`, index `0` pairs with `64`.
- RoPE applies to Q and K only. V is cached and attended to but never rotated.
- In cached decode, only the newly written K rows are rotated. Old cached K rows were already rotated when first written.
- RMSNorm is not LayerNorm: no mean subtraction, no bias. It normalizes by root mean square and applies gamma with epsilon inside the square root.
- Each decoder layer has two distinct RMSNorm gammas: `input_layernorm.weight` before attention and `post_attention_layernorm.weight` before FFN. The final norm uses `model.norm.weight`.
- Checkpoint projection weights are `[out,in]`; runtime matmul wants `[in,out]`; this repo transposes once at load time.
- The assignment text says `lm_head` is shared with the embedding table, but this repo's checkpoint has `tie_word_embeddings=false`.
- For this repo, logits must use `global/lm_head_weight.bin`, not the embedding table.
- `lm_head` is a separate `[VOCAB_SIZE, EMBEDDING_DIM]` tensor. With repo constants, that is `[128256,4096]`.
- Final RMSNorm is applied to all current rows, then only the last hidden row per batch slot is copied for logits.
- `compute_lm_head_logits` is a CPU host loop in this repo, not a GPU kernel.
- Packed Q is `[s,4096] = [s,32*128]`.
- Packed K and V are `[s,1024] = [s,8*128]`.
- GQA maps query head `hi` to KV group `kvg = hi / 4`; four Q heads share one K/V head.
- The repo reuses one scratch score matrix `scratch.d_S [q_seq,kv_seq]` inside a per-head loop. Do not depict 32 score matrices allocated at once as the implementation.
- Batched resident generation requires equal tokenized prompt lengths; `validate_equal_lengths(...)` rejects mixed-length batches.
- Exact constants to preserve if shown: `500000`, `1e-5`, `128009`, `128256`, `4096`, `1024`, `14336`, `32`, `8`, `4`, `128`, `-1e6`.
- Source anchors for factual grounding include `docs/assignment/llm_part2.md`, `reference.py`, `tools/verify_reference.py`, split `tests/test_m2m3_*.cpp` files, `assets/llama3/config.json`, `docs/learnings.md`, `config.h`, `kernel/rmsnorm.cu`, `kernel/rope.cu`, `kernel/attention.cu`, `kernel/swiglu.cu`, `kernel/residual.cu`, `src/inference_layer.cu`, `src/inference_loop.cu`, `src/model_weights.cpp`, `include/model_weights.h`, and `include/inference.h`.
