Core story:
Grouped Query Attention keeps 32 query heads, but shares only 8 key/value
heads, so every 4 query heads reuse one K/V group. For one query head, attention
scores Q against the matching K group, blocks future key positions when doing
square prefill, turns scores into stable softmax weights, and uses those weights
to mix the matching V group. The 32 per-head outputs are stitched back into a
4096-wide attention result; output projection and residual happen after this
diagram.

Guardrail facts the image must obey but should not render unless listed above:
- Llama 3 uses GQA, not standard MHA.
- `NUM_HEADS = 32`, `NUM_KV_HEADS = 8`, `HEAD_DIM = 128`,
  `EMBEDDING_DIM = 4096`.
- `KVCache::kv_dim() = NUM_KV_HEADS * HEAD_DIM = 1024`.
- Q has width `32 * 128 = 4096`; K and V have width `8 * 128 = 1024`.
- For query head `i`, the K/V group is
  `g = floor(i / (h / h_k)) = floor(i / 4)`.
- Query head `i` compares with K from the same decoder layer's current/cached
  token positions, not with K from previous layers.
- RoPE happens before attention and only affects Q and K. V is not rotated.
- K and V are written into the KV cache during projection. RoPE is applied to
  the newly written K rows. Attention reads cached K/V over the valid prefix.
- Attention reads cache rows `[0, kv_seq)`, not the full allocated `S_MAX`
  capacity.
- `kv_seq = len_before + q_seq`.
- Assignment prefill shape is `[s,s]`.
- Repo cached attention shape is `[q_seq,kv_seq]`.
- Per-head score shape is
  `Q_i [q_seq,128] * K_g^T [128,kv_seq] -> S_i [q_seq,kv_seq]`.
- Scale is exactly `1 / sqrt(HEAD_DIM) = 1 / sqrt(128)`.
- The causal mask sets future key columns `p > q` to a large negative sentinel
  (repo kernel condition: `col > row`).
- Mask axes: `q` is the query row on the vertical axis; `p` is the key-position
  column on the horizontal axis. The red masked region is above the diagonal.
- The assignment describes `-inf`; this repo's kernel writes `-1e6f`.
- `run_attention_heads` applies `gpu_causal_mask` only when
  `q_seq == kv_seq && q_seq > 1`, the square prefill case.
- Decode uses `q_seq = 1` and skips the square causal mask because the one new
  query row attends only over the already-valid cached prefix.
- Do not draw an upper-triangle mask on rectangular decode attention
  `[1,kv_seq]`.
- Stable softmax subtracts the row maximum before exponentiation:
  `exp(S[q,p] - row_max) / sum_k exp(S[q,k] - row_max)`.
- `O_i = alpha_i * V_g` has shape `[q_seq,128]`.
- Concatenating all 32 `O_i` blocks produces `O [q_seq,4096]`.
- GQA returns the stitched attention result. Output projection `W_O`, residual,
  post-attention RMSNorm, FFN, and the next residual happen after this.
- `run_attention_heads` loops over all 32 query heads for one batch slot at a
  time.
- One `AttentionScratch` object is allocated per `forward_step` and reused for
  every layer, batch slot, and head:
  `d_Qi [q_seq,128]`, `d_KgT [128,kv_seq]`, `d_Vg [kv_seq,128]`,
  `d_S [q_seq,kv_seq]`, `d_Oi [q_seq,128]`.
- This implementation reuses one scratch score buffer `scratch.d_S`; do not
  depict 32 simultaneous score matrices as the repo implementation.
- The assignment may discuss all 32 score matrices at once as a thought
  experiment; that is not how this implementation allocates scores.
- `gpu_gather_head` slices Q and V.
- `gpu_gather_head_transpose` slices K and writes `K_g^T`.
- `gpu_matmul_device` performs `Q_i * K_g^T` and `softmax(S) * V_g`.
- `gpu_scale`, `gpu_causal_mask`, `gpu_softmax`, and `gpu_scatter_head` perform
  the other per-head attention steps.
- Gather/scatter stays on GPU; do not imply host round trips for per-head
  slices.
- Current generation uses full-prompt prefill and then one-token decode steps.
- Multi-token generation is supported as a loop of one-token decode steps. Do not
  imply one decode `forward_step` processes multiple new tokens at once
  (`q_seq > 1`), and do not imply mixed-length batched prompts are supported.
