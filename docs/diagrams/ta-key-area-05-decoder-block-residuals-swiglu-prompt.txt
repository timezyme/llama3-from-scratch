Title

Key Area 5: Decoder Block = Two Residual Updates

Knowledge-transfer goal

Create a clean 16:9 CS265 TA-prep infographic that teaches the decoder block story in 10 seconds. A student should be able to explain it out loud in under 60 seconds: one layer keeps the residual stream shape, adds an attention update, then adds a SwiGLU FFN update.

Core story

One decoder block is two "keep X, add an update" moves. First, attention gathers cross-token context and projects it back to the residual-stream shape, so `X = X + attn_out` is valid. Second, after a separate post-attention RMSNorm, the FFN expands each token row to a larger SwiGLU space, gates it, projects it back to the residual-stream shape, then adds it with `X = X + ffn_out`. The block repeats 32 times, and the shape returns to `[rows,4096]` after each update.

Guardrail facts the image must obey but should not render unless listed

- Assignment block order: `RMSNorm -> Q/K/V projections -> RoPE on Q,K -> Attention -> Output projection -> Residual -> RMSNorm -> FFN -> Residual`, repeated for `NUM_LAYERS = 32`.
- Repo implementation loop is in `forward_step` in `src/inference_layer.cu`: `for (int layer = 0; layer < NUM_LAYERS; ++layer)`.
- Repo code uses `rows = batch * q_seq`; the assignment mental model is `[s,4096]`.
- Constants: `EMBEDDING_DIM = 4096`, `FFN_DIM = 14336`, `NUM_LAYERS = 32`, `RMS_NORM_EPSILON = 1e-5`.
- Q, K, and V are separate projection matmuls using `W_Q`, `W_K`, and `W_V`; never show a fused `W_qkv`.
- RoPE rotates Q and the newly written K rows only. V is not rotated.
- Attention update: stitched attention output `O` is projected with `W_O` to `attn_out [rows,4096]`, then residual #1 applies `X = X + attn_out`.
- Residual #1 happens only after the output projection returns the attention update to `[rows,4096]`.
- FFN update uses the second RMSNorm weight, `post_attention_layernorm.weight`, distinct from `input_layernorm.weight`. This second RMSNorm happens after residual #1 and before `gate`/`up`.
- FFN formulas: `gate = X_norm * W_gate^T`, `up = X_norm * W_up^T`, `H = SiLU(gate) * up`, `ffn_out = H * W_down^T`, then `X = X + ffn_out`.
- `gate` and `up` have shape `[rows,14336]`; `ffn_out` returns to `[rows,4096]`.
- `gpu_swiglu(d_gate, d_up, d_gate, rows * FFN_DIM)` computes `SiLU(gate) * up` and aliases output back into `d_gate` in this repo.
- Residual calls use `gpu_residual_add(d_X, d_attn_out, rows * d)` and `gpu_residual_add(d_X, d_ffn, rows * d)`.
- `kernel/residual.cu` implements elementwise `a[i] += b[i]`.
- Assignment formulas use `X * W^T`. This repo transposes projection weights at load time in `src/model_weights.cpp`, so runtime matmul receives already-transposed `[in,out]` buffers such as `d_wgate [4096,14336]`, `d_wup [4096,14336]`, and `d_wdown [14336,4096]`.
- Final RMSNorm happens once after all 32 decoder blocks, before the output layer. It is not part of this repeated block.
- Repo-specific output-layer exception: `tie_word_embeddings=false`; logits use a separate `lm_head` weight. Do not render output-layer details in this image.
